# 高级数据分析方法与实现

> 面向论文写作的技术说明文档。对应本仓库 `src/analysis.py` 中的
> `run_statistical_analysis()`、`run_structure_analysis()`、`run_compliance_analysis()`
> 三段新增分析，以及 `run_advanced_analysis()` 一键汇总入口。所有模块以 Spark 3.x
> SQL 实现，与已有的 K-Means 聚类 DWS 层并列，共同支撑 Flask 看板的「高级分析」页。

---

## 1 研究背景与动机

原有分析链路在对清洗后的 Parquet（12.5 万条医美项目销售记录、20 座城市、
5 个品类、1500 家机构、2 个采集月份）之上，只做了两类计算：

1. **多维 `GROUP BY` 聚合**——按城市 / 品类 / 渠道 / 医生头衔 / 折扣分桶等
   维度的销量合计、均价、均评分统计；
2. **K-Means 聚类 + 8 张派生主题宽表（DWS 层）**——利用窗口函数与 CTE
   对聚类结果进行画像、偏好度、排名、月度环比等二次加工。

这两类计算都是「把行按维度分组再求和、求平均」，**无法回答量化关系层面的
业务问题**：

- 价格与销量之间的相关强度是多少？价格弹性在不同品类间有没有差异？
- 医美市场是否存在垄断倾向？头部机构的市场份额是多少？
- 哪些机构存在合规风险？风险从何处量化？

为此本次迭代在 K-Means DWS 层之外又扩展了**一组统计 / 结构 / 合规导向
的分析函数**，补齐 `GROUP BY` 触及不到的维度。

## 2 方法论总览

本章集中给出后续三大模块依赖的 6 个数理定义。

### 2.1 Pearson 相关系数

对两列数值 $X,Y$ 的样本，定义

$$
r_P(X,Y) \;=\; \frac{\operatorname{cov}(X,Y)}{\sigma_X \sigma_Y}
\;=\; \frac{\sum_{i}(x_i-\bar x)(y_i-\bar y)}
              {\sqrt{\sum_i(x_i-\bar x)^2}\,\sqrt{\sum_i(y_i-\bar y)^2}}.
$$

Spark SQL 提供内置聚合 `corr(X, Y)` 直接计算 $r_P$。

### 2.2 Spearman 等级相关系数

令 $R_X(i), R_Y(i)$ 为第 $i$ 个样本在列 $X, Y$ 上的升序排名，则

$$
r_S(X,Y) \;=\; r_P\bigl(R_X,\; R_Y\bigr).
$$

即「排名后求 Pearson」。Spark SQL 没有内置 `spearman_corr`，
本实现用窗口函数 `RANK() OVER (ORDER BY col)` 派生排名列，再套用 `corr`。

### 2.3 一元线性回归与价格弹性

令 $p$ 为到手价、$q$ 为月销量。对数双侧回归

$$
\ln q_i \;=\; \alpha + \beta\,\ln p_i + \varepsilon_i,
$$

其中系数 $\beta = \dfrac{\partial \ln q}{\partial \ln p}$ 即**价格弹性**。
当 $\beta<-1$ 时称为「弹性」；$-1<\beta<0$ 为「缺乏弹性」；
$\beta\approx 0$ 表示价格变化对需求几乎无影响。

Spark SQL 内置 `regr_slope(y, x)`、`regr_intercept(y, x)`、`regr_r2(y, x)`
三个聚合函数，按品类分组即可一次性得到各品类的 $(\hat\alpha,\hat\beta,R^2,n)$。

### 2.4 Herfindahl-Hirschman Index（HHI）

设某市场内有 $n$ 家机构，机构 $i$ 的市场份额 $p_i = s_i / \sum_j s_j$
（这里 $s_i$ 取该机构的月销量合计），则

$$
\mathrm{HHI} \;=\; \Bigl(\sum_{i=1}^{n} p_i^{2}\Bigr)\times 10\,000.
$$

美国司法部《横向并购指南》的经验阈值：

| HHI 区间 | 集中度判断 |
|---|---|
| `< 1500` | 竞争充分（unconcentrated） |
| `1500 – 2500` | 中度集中（moderately concentrated） |
| `> 2500` | 高度集中（highly concentrated） |

派生指标：前 $n$ 家集中度 $\mathrm{CR}_n = \sum_{i\le n} p_i$。

### 2.5 帕累托 20/80（累计分布）

将全部项目按月销量降序排列，记
$\mathrm{cum\_share}(k) = \sum_{i\le k} s_i \big/ \sum_i s_i$
与 $\mathrm{project\_share}(k) = k/N$。若 $\mathrm{cum\_share}$ 在
$\mathrm{project\_share}=0.2$ 处已达 0.8，即满足经典 80/20。

### 2.6 余弦相似度

对城市 $c$ 按品类 $t$ 的销量向量
$\mathbf{v}_c=(s_{c,t_1},\dots,s_{c,t_m})$，两城市的结构相似度

$$
\cos(\mathbf{v}_a,\mathbf{v}_b) \;=\;
\frac{\mathbf{v}_a\cdot\mathbf{v}_b}{\|\mathbf{v}_a\|\,\|\mathbf{v}_b\|}
\in[-1,1].
$$

医美销量天然非负，故实际结果落在 $[0,1]$。值越接近 1 表示两城市的品类
消费结构越相似。

## 4 模块一：统计检验与相关性

代码入口：`src/analysis.py::run_statistical_analysis()`  
产物：`output/advanced_tables/stat_*.parquet` + `output/statistical_analysis.json`

### 4.1 Pearson / Spearman 相关矩阵

参与矩阵的 6 个数值特征：

`list_price`、`actual_price`、`discount_rate`、`monthly_sales`、
`review_count`、`rating`。

**实现要点**：$N=6$ 列两两相关共需 $\binom{6}{2}=15$ 个系数。直接写
15 个 `SELECT corr(a,b) FROM t` 会让 Spark 扫表 15 次，成本极高。改为**合并
成一次 SELECT**，让物理计划共享一次全表聚合：

```sql
SELECT
  corr(list_price,   actual_price)  AS c__0__1,
  corr(list_price,   discount_rate) AS c__0__2,
  corr(list_price,   monthly_sales) AS c__0__3,
  ...
  corr(review_count, rating)        AS c__4__5
FROM sales_advanced
```

Python 端遍历 `itertools`-风格的下标对，把单行宽表再展开为 $N\times N$
对称矩阵以便前端热力图渲染：

```python
def _corr_matrix(view_name, col_ref):
    parts, pairs = [], []
    for i, a in enumerate(available):
        for b in available[i+1:]:
            alias = f"c__{i}__{available.index(b)}"
            parts.append(f"corr({col_ref(a)}, {col_ref(b)}) AS {alias}")
            pairs.append((alias, a, b))
    row = spark.sql("SELECT " + ", ".join(parts) + " FROM " + view_name).collect()[0]
    out = [{"feature_a": a, "feature_b": a, "corr": 1.0} for a in available]
    for alias, a, b in pairs:
        v = row[alias]
        val = float(v) if v is not None else None
        out.extend([
            {"feature_a": a, "feature_b": b, "corr": val},
            {"feature_a": b, "feature_b": a, "corr": val},  # 对称补齐
        ])
    return out
```

**Spearman** 复用同一个 `_corr_matrix`，只是先用窗口函数构造一张排名视图
`sales_ranked`：

```sql
SELECT
  RANK() OVER (ORDER BY list_price)    AS r_list_price,
  RANK() OVER (ORDER BY actual_price)  AS r_actual_price,
  ...
FROM sales_advanced
WHERE list_price IS NOT NULL AND actual_price IS NOT NULL AND ...
```

### 4.2 分品类价格弹性回归

```sql
WITH base AS (
  SELECT category,
         LN(actual_price)  AS ln_price,
         LN(monthly_sales) AS ln_sales
  FROM sales_advanced
  WHERE category IS NOT NULL AND actual_price > 0 AND monthly_sales > 0
)
SELECT category,
       regr_slope(ln_sales,  ln_price) AS elasticity,
       regr_intercept(ln_sales, ln_price) AS intercept,
       regr_r2(ln_sales,   ln_price) AS r2,
       COUNT(*) AS n
FROM base
GROUP BY category
ORDER BY elasticity ASC NULLS LAST
```

前端在条形图上按 $\hat\beta$ 符号 / 幅度染色（红：弹性；橙：缺乏弹性；
绿：正弹性，通常出现在奢侈品信号效应或数据噪声）。

### 4.3 评分-销量分层相关

`HAVING COUNT(*) >= 30` 过滤样本过小的分组，避免系数不稳定：

```sql
SELECT city AS city,
       corr(rating, monthly_sales) AS corr_rating_sales,
       COUNT(*) AS n, AVG(rating) AS avg_rating, AVG(monthly_sales) AS avg_sales
FROM sales_advanced WHERE city IS NOT NULL
GROUP BY city HAVING COUNT(*) >= 30
ORDER BY corr_rating_sales DESC NULLS LAST
```

按品类版本结构相同。

## 5 模块二：关联与结构分析

代码入口：`src/analysis.py::run_structure_analysis()`  
产物：`output/advanced_tables/struct_*.parquet` + `output/structure_analysis.json`

### 5.1 机构集中度 HHI + CR10

实现采用**三级 CTE**，一次 SQL 即出 HHI / CR5 / CR10：

```sql
WITH inst_sales AS (                                      -- ① 先把机构销量聚合
  SELECT institution, SUM(monthly_sales) AS ss
  FROM sales_advanced WHERE institution IS NOT NULL
  GROUP BY institution
),
with_total AS (                                           -- ② 窗口 SUM 求总量 + 排名
  SELECT institution, ss,
         SUM(ss) OVER ()                           AS total_ss,
         ROW_NUMBER() OVER (ORDER BY ss DESC)      AS rn
  FROM inst_sales
)
SELECT                                                    -- ③ 一次性聚合出 HHI/CR
  SUM(POWER(ss / total_ss, 2)) * 10000                           AS hhi,
  SUM(CASE WHEN rn <= 10 THEN ss / total_ss END)                 AS cr10,
  SUM(CASE WHEN rn <= 5  THEN ss / total_ss END)                 AS cr5,
  COUNT(*)                                                       AS institution_count,
  MAX(total_ss)                                                  AS total_sales
FROM with_total
```

**按城市 / 按品类版本**把上述 CTE 的每层都加上 `PARTITION BY city`
或 `PARTITION BY category`，并在最外层同时输出集中度等级：

```sql
CASE
  WHEN hhi < 1500 THEN '竞争充分'
  WHEN hhi < 2500 THEN '中度集中'
  ELSE '高度集中'
END AS concentration_level
```

### 5.2 项目销量帕累托 20/80

核心是「累计窗口」的 `SUM ... OVER (ORDER BY ... ROWS BETWEEN UNBOUNDED
PRECEDING AND CURRENT ROW)`：

```sql
WITH proj AS (
  SELECT project_name, SUM(monthly_sales) AS total_sales
  FROM sales_advanced WHERE project_name IS NOT NULL
  GROUP BY project_name
),
ranked AS (
  SELECT project_name, total_sales,
         ROW_NUMBER() OVER (ORDER BY total_sales DESC) AS rn,
         SUM(total_sales) OVER (ORDER BY total_sales DESC
             ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW) AS cum_sales,
         SUM(total_sales) OVER () AS grand_sales,
         COUNT(*)         OVER () AS grand_count
  FROM proj
)
SELECT project_name, total_sales, rn, cum_sales,
       cum_sales / grand_sales         AS cum_share,
       CAST(rn AS DOUBLE) / grand_count AS project_share
FROM ranked
ORDER BY rn
```

Python 端取出**50% / 80% / 95%** 三个关键切点，用于前端折线图的标注线：

```python
def _first_reach(rows, threshold):
    for row in rows:
        if row.get("cum_share") is not None and row["cum_share"] >= threshold:
            return {"cum_share_threshold": threshold,
                    "project_share": row["project_share"], "rn": row["rn"]}
    return None
breakpoints = [_first_reach(rows, t) for t in (0.5, 0.8, 0.95)]
```

### 5.3 城市消费结构余弦相似度

余弦相似度矩阵是稠密的 $n\times n$（$n=20$ 个城市时不大），但做法是
**Spark 负责聚合、pandas + numpy 负责矩阵运算**——扬长避短：

```python
cc_df = spark.sql("""
    SELECT city, category, SUM(monthly_sales) AS total_sales
    FROM sales_advanced
    WHERE city IS NOT NULL AND category IS NOT NULL
    GROUP BY city, category
""")
pdf = cc_df.toPandas()
mat = pdf.pivot_table(index="city", columns="category",
                      values="total_sales", fill_value=0)
arr = mat.values.astype(float)
norms = np.linalg.norm(arr, axis=1, keepdims=True); norms[norms == 0] = 1.0
sim = (arr / norms) @ (arr / norms).T            # 20×20 对称矩阵
```

除完整矩阵外，额外为每个城市保留 Top-5 相似城市清单，方便前端展示。

## 6 模块三：合规风险打分

代码入口：`src/analysis.py::run_compliance_analysis()`  
产物：`output/advanced_tables/comp_*.parquet` + `output/compliance_analysis.json`

本模块首次引入此前未被使用的三个合规字段：`approval_no`（批准文号）、
`device_brand`（器械品牌）、`udi_match`（UDI 匹配标志）。

### 6.1 四项合规指标

在一次聚合中并行产出 4 个比例型指标 $\in [0,1]$：

| 指标 | 定义 |
|---|---|
| **UDI 不匹配率** `udi_mismatch_rate` | `AVG(CASE WHEN udi_match IS NULL OR udi_match=0 THEN 1.0 ELSE 0.0 END)` |
| **疑似无效批准文号率** `invalid_approval_rate` | `AVG(CASE WHEN approval_no IS NULL OR approval_no IN ('未知','','nan','None') OR approval_no NOT LIKE '国械%' THEN 1.0 ELSE 0.0 END)` |
| **低评分率** `low_rating_rate` | `AVG(CASE WHEN rating < 4.0 THEN 1.0 ELSE 0.0 END)` |
| **价格异常率** `price_anomaly_rate` | 相对全局 $\mu,\sigma$ 的 z-score 外三倍标准差：`AVG(CASE WHEN σ>0 AND ABS((actual_price-μ)/σ)>3 THEN 1.0 ELSE 0.0 END)` |

价格基线 $(\mu,\sigma)$ 先用一条独立 SQL 预计算一次，再把数字以字面量
回填到主查询中，避免在每行做全局窗口。

### 6.2 机构风险分与分级

按经验权重合成 0–100 风险分：

$$
\mathrm{RiskScore}
  \;=\; 100\times\bigl(0.40\,u + 0.30\,a + 0.20\,r + 0.10\,z\bigr)
$$

其中 $u,a,r,z$ 依次为上表四个比例。权重设计思路：

- UDI 不匹配直接关系器械可追溯性，权重最高；
- 批准文号缺失 / 异常次之；
- 低评分 / 价格异常作为辅助信号，反映服务质量与潜在刷单。

配合 `RANK() OVER (ORDER BY risk_score DESC)` 输出机构风险排名和分级：

```sql
CASE
  WHEN risk_score >= 30 THEN '高风险'
  WHEN risk_score >= 15 THEN '中风险'
  ELSE '低风险'
END AS risk_level
```

为避免小样本机构的指标方差过大，主查询用
`HAVING COUNT(*) >= 5` 过滤掉极小样本机构。

### 6.3 品类 / 城市合规画像

机构维度之外，同一组 4 个比例指标也按品类、城市各输出一张表，
用于发现**结构化合规短板**（某品类或某城市系统性地 UDI 不达标 / 评分
偏低）。

## 7 实验结果（基于当前合成数据集）

数据集：12.5 万条医美项目销售记录 × 20 城市 × 5 品类（外科 / 光电 / 注射 /
口腔 / 皮肤管理）× 1500 机构 × 2 个采集月份（2025-08、2025-09）。

### 7.1 相关性矩阵（Pearson，部分关键对）

| 特征 A | 特征 B | $r_P$ | 解读 |
|---|---|---|---|
| list_price | actual_price | **0.962** | 标价与到手价强正相关（符合预期：折扣率波动有限） |
| discount_rate | monthly_sales | $-5.4\times10^{-4}$ | 折扣与销量几近独立 |
| rating | monthly_sales | $-4.1\times10^{-3}$ | 评分与销量几近独立 |
| review_count | monthly_sales | ≈ 0 | 评价数与销量几近独立 |

> 除 `list_price ↔ actual_price` 外，其余配对均接近 0。考虑到该数据
> 为合成数据集，价格 / 销量 / 评分三者在生成时相互独立，结果与先验一致。
> 真实数据场景下预计会观察到显著非零的 $r_P(\text{price}, \text{sales})$
> 与 $r_P(\text{rating}, \text{sales})$。

### 7.2 分品类价格弹性

| 品类 | $\hat\beta$ | $R^2$ | $n$ |
|---|---:|---:|---:|
| 皮肤管理 | $-0.0144$ | $9.9\times10^{-5}$ | 28 881 |
| 注射 | $+0.0060$ | $1.5\times10^{-5}$ | 29 157 |
| 口腔 | $+0.0066$ | $5.2\times10^{-5}$ | 20 113 |
| 光电 | $+0.0078$ | $6.9\times10^{-5}$ | 26 803 |
| 外科 | $+0.0173$ | $8.9\times10^{-5}$ | 19 351 |

所有品类 $|\hat\beta|<0.02$ 且 $R^2<10^{-3}$，表明合成数据中对数价格
对对数销量几乎没有解释力，不支持传统需求曲线。

### 7.3 机构集中度

**全局**：$\mathrm{HHI}=6.78$，$\mathrm{CR}_5=0.48\%$，
$\mathrm{CR}_{10}=0.94\%$，$n=1500$，$\sum s=1.496\times10^{7}$。

**按城市 Top 5**（按 HHI 降序）：

| 城市 | HHI | CR10 | 机构数 | 集中度等级 |
|---|---:|---:|---:|---|
| 合肥 | 136.4 | 16.48% | 75 | 竞争充分 |
| 深圳 | 136.3 | 16.68% | 75 | 竞争充分 |
| 西安 | 136.0 | 16.79% | 75 | 竞争充分 |
| 南京 | 136.0 | 16.45% | 75 | 竞争充分 |
| 宁波 | 135.7 | 16.51% | 75 | 竞争充分 |

**按品类**：5 大品类 HHI 均在 $7.11\sim7.38$ 之间，属极端竞争充分状态。

### 7.4 项目帕累托

| 累计销量阈值 | 项目占比 | 项目绝对数量 |
|---|---:|---:|
| 50% | 35.6% | 16 / 45 |
| 80% | 66.7% | 30 / 45 |
| 95% | 84.4% | 38 / 45 |

比经典 80/20 更均匀，与合成数据假设下销量近似服从窄分布、
头部效应弱相符。

### 7.5 城市消费结构相似度

20 座城市两两余弦相似度全部 $>0.999$，说明在合成数据中每座城市的
5 个品类占比几乎相同，城市间没有结构差异。该指标在真实数据中将
用于识别「消费结构相似」的城市集群，支撑跨区域投放 / 选址决策。

### 7.6 合规风险打分

**全局价格基线**：$\mu = 3473.25$ 元、$\sigma = 3709.68$ 元。

**品类合规画像**：UDI 不匹配率在 9.67%–10.09% 之间，低评分率在
9.11%–9.85% 之间，各品类差异 $<1$ 个百分点。

**机构风险分布**：$n=1500$ 家机构全部落入「低风险」区间
（$\max \mathrm{RiskScore}=12.19$，阈值 15），排名前 3 如下：

## 9 技术创新小结

1. **与现有 DWS-A（聚类主题表）并列的第二条 DWS 链路**。DWS-A 切片维度是
   K-Means 聚类结果，DWS-B 切片维度是统计学 / 经济学 / 合规工程；二者都
   物化为独立 Parquet，保证了数仓分层的清晰。
2. **相关矩阵的一次性聚合**。把 $N(N-1)/2$ 个两两 `corr` 合并到单条
   `SELECT`，物理扫表从 $\mathcal O(N^2)$ 次降到 1 次。
3. **帕累托累计分布的窗口函数写法**。`SUM ... OVER (... ROWS BETWEEN
   UNBOUNDED PRECEDING AND CURRENT ROW)` 在 Spark 里一次得到累计销量、
   累计占比、项目占比三列，相比循环扫描的命令式写法更符合声明式数据
   仓库的表达习惯。
4. **Spark + pandas 混合计算**。城市×品类相似度矩阵是稠密小矩阵，
   在 20×5 规模下 pandas pivot + numpy 矩阵乘法远快于 Spark 的
   `CROSS JOIN`。分层合作保留 Spark 的聚合扩展性、又兼顾了密集线性代数。
5. **多维度合规风险合成**。给医美行业关键合规字段
   (`approval_no` / `udi_match`) 首次赋予量化权重，输出 0-100 分与
   三档风险等级，具备直接产品化价值。
6. **声明式阈值判定**。HHI 的「竞争充分 / 中度集中 / 高度集中」与
   风险等级的「高 / 中 / 低」均通过 `CASE WHEN` 在 SQL 层完成，
   前端只负责展示，减少客户端逻辑。
