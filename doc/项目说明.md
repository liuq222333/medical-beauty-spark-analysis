# 基于Spark的医美数据分析系统 — 项目说明

## 一、项目概述

本课题为**基于Spark的医美数据分析系统的设计与实现**，属于大数据应用方向的毕业设计。系统面向医美行业，对医美项目销售数据进行采集、清洗、存储、聚类分析与可视化展示，为医美平台提供数据支撑，辅助制定运营与改进策略，提升工作效率与销售转化。

系统采用Apache Spark作为大数据计算框架，完成从原始数据到分析结果的全流程处理，并按照软件工程规范完成系统分析、设计、实现与测试，同时撰写符合沈阳工学院要求的毕业论文。

---

## 二、设计目的

1. **数据清洗与预处理**  
   对收集的医美项目销售数据进行清洗和预处理，包括：  
   - 去除重复数据  
   - 处理缺失值  
   - 处理异常值  
   保证后续分析所依赖的数据质量。

2. **聚类分析与策略制定**  
   对项目数据进行聚类分析，识别不同项目、地区、渠道下的销售特征与趋势，为运营改进、选品与定价等提供依据。

3. **趋势与消费预测**  
   通过对地区医美项目销售趋势的分析，识别顾客可能消费的项目类型与偏好，为精准营销与资源调配提供数据支持。

## 四、设计思路与模块划分

系统按功能划分为以下四个模块：

### 1. 数据采集与预处理模块

- **数据采集**：对医美项目销售相关数据进行收集（如平台、城市、商圈、机构、项目品类、价格、销量、评价等）。
- **数据清洗**：  
  - 检测并处理重复记录  
  - 识别并处理缺失值（填充或剔除策略）  
  - 识别并处理异常值（如价格、销量、评分的异常）  
- **数据整理**：统一字段格式、编码与单位，为存储与分析提供规范数据。

### 2. 数据存储模块

- 将清洗后的数据持久化到数据库中，保证数据完整性与可追溯性。
- 设计合理的表结构（或宽表/分区策略），支持按地区、时间、项目类型等维度的查询与统计，满足分析模块和可视化模块的访问需求。

### 3. 数据分析模块

- 基于Spark实现**聚类分析**（如K-Means等），对医美项目销售数据进行分群，发现：  
  - 项目组合与销售表现的关系  
  - 不同地区、渠道的销售特征  
- 识别医美项目的**销售趋势**（如品类热度、区域偏好、价格带分布等），为医美平台提供数据支持，进而支撑选品、定价与促销策略，增加销售量。

#### 3.1 数据仓库分层设计（ODS → DWD → DWS → ADS）

为避免分析层仅停留在"对清洗后数据跑简单聚合 SQL"的初级水平，本系统将 K-Means 聚类结果作为一级数据产物，**再派生多张主题宽表**（数仓 DWS 层），在 Spark SQL 中使用 CTE、窗口函数、多表 JOIN、多条件 CASE WHEN 等更丰富的构造方式，使分析具有业务语义且可扩展。

分层示意：

```
ODS 层（原始数据）       data/medical_beauty_data.csv（UTF-8，英文列名）
    ↓ preprocess（pandas：去重 / 缺失值 / IQR 异常值 / 日期修复 / 统计月份派生）
DWD 层（明细）            output/clean_data.parquet
    ↓ K-Means 聚类（Spark MLlib：VectorAssembler + StandardScaler + KMeans，K=3）
DWD+ 层（含聚类标签）     output/clustered_data.parquet  +  output/cluster_centers.json
    ↓ 聚类主题派生（Spark SQL：CTE + 窗口函数 + 多表 JOIN）
DWS 层（主题宽表）        output/cluster_tables/*.parquet（共 8 张）
    ↓ 聚合物化
ADS 层（应用数据）        output/trend_stats.json  +  output/cluster_insights.json
    ↓ 看板读取
可视化                   /api/analytics、/api/cluster、/api/cluster_insights
```

#### 3.2 聚类派生主题表（DWS 层，共 8 张）

每张表均由一段 Spark SQL 生成，物化为独立 Parquet 文件；聚合后的结果同时写入 `cluster_insights.json` 供 Web 看板的"聚类深度分析"页直接读取。

| 表名 | 主题 | 核心技术点 |
|------|------|------------|
| `cluster_profile` | 聚类画像 + 业务标签 | 两级 CTE；`RANK() OVER` 计算价格/销量排名；多条件 `CASE WHEN` 自动打"高端精品型 / 大众爆款型 / 长尾引流型 / 中端主力型"业务标签，并按均折扣分"深折促销 / 中度折扣 / 高折扣/原价"三档 |
| `cluster_city_rank` | 聚类×城市销量分布 | `ROW_NUMBER() OVER (PARTITION BY cluster ORDER BY total_sales DESC)` 给出各聚类内城市排名；`RANK() OVER (PARTITION BY city)` 给出同城市内各聚类排名 |
| `cluster_category_preference` | 聚类品类偏好度 Lift | 4 级 CTE（cluster_cat / cluster_total / global_cat / global_total）+ 多表 JOIN + CROSS JOIN；计算偏好系数 Lift = 聚类内品类占比 ÷ 全局品类占比，>1 表示该聚类偏好该品类 |
| `cluster_channel_efficiency` | 聚类×渠道效率 | 加权销量 `SUM(销量 × 评分)`；双向 `RANK`（簇内渠道排名 + 渠道内簇排名） |
| `cluster_price_band` | 聚类×折扣分桶 | 多条件 `CASE WHEN` 分 5 档折扣；`PERCENT_RANK() OVER (PARTITION BY cluster)` 计算簇内销量百分位 |
| `cluster_top_projects` | 聚类内 TOP 5 项目 | `ROW_NUMBER() OVER` + `WHERE rn <= 5` 取每聚类销量前 5 的热销项目 |
| `cluster_doctor_matching` | 聚类×医生头衔匹配 | `COUNT DISTINCT` 统计不重复医生数；`RANK()` 簇内医生头衔销量排名 |
| `cluster_month_delta` | 聚类月度销量环比 | 两级 CTE；`LAG(total_sales) OVER (PARTITION BY cluster ORDER BY stats_month)` 取上月销量；`COALESCE` 处理首月空值；计算销量绝对增量与增长率 |

#### 3.3 分析技术栈总结

| 层级 | 框架 / 技术 |
|------|------------|
| 清洗 | pandas（避免 Windows 下 PySpark Worker 连接超时） |
| 存储 | Parquet（本地）/ HDFS（可选，通过 `USE_HDFS=1` 切换）+ pyarrow |
| 聚类 | PySpark MLlib：`VectorAssembler` + `StandardScaler` + `KMeans`（K=3） |
| 趋势分析 | Spark SQL：基础聚合 15 段 + 聚类派生 8 段 |
| 看板 | Flask + ECharts，REST API 返回 JSON |

### 4. 数据可视化模块

- 将聚类结果、趋势统计、地区对比等分析结果以图表形式直观展示（如柱状图、饼图等）。
- 看板提供地区销量、品类分布、渠道分布及聚类结果说明，便于业务人员理解与决策。

**数据可视化模块说明（与当前实现一致，可用于论文/答辩）：**

数据可视化模块将聚类结果与趋势统计数据以图表形式直观呈现，是系统面向最终用户的核心交互界面。钱倩文等学者对医美用户信息行为的研究表明，直观易懂的数据呈现方式对于辅助用户理解与决策具有重要价值<sup>[5]</sup>。陆嘉欣对医疗美容医院营销策略的研究也指出，数据驱动的可视化分析工具是提升医美机构运营决策水平的有效手段<sup>[8]</sup>。

本模块主要包含以下可视化图表设计：

- **地区销量 TOP 图**：采用柱状图展示各城市的销售总量排行，便于快速识别销量领先区域，为区域运营与资源投放提供依据。
- **项目品类销量分布图**：采用饼图展示各医美品类的销量占比，直观呈现品类结构，辅助选品与品类策略分析。
- **渠道销量分布图**：采用柱状图展示各渠道（如线上、线下等）的销量对比，支撑渠道效果评估与营销策略调整。
- **聚类结果说明**：以文字形式展示 K-Means 聚类数 K 及所用特征维度（如标价、到手价、折扣率、月销量、评价数、评分），说明聚类分析结论，便于业务人员理解分群含义。

所有图表均支持鼠标悬浮查看详细数据，整体界面布局清晰，操作便捷，满足业务人员日常数据分析与决策支持的需求。
